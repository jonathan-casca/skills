# Performance And Scale Prompts

```text
Perform a performance review of this codebase. Identify slow paths, N+1 queries, missing indexes, large payloads, unnecessary re-renders, unbounded background work, serial external calls, cache opportunities, and expensive startup behavior.
```

```text
Create a load testing plan for this app. Define critical user journeys, target concurrency, data volume, success criteria, setup data, measurement points, failure thresholds, and how to analyze results.
```

```text
Audit database performance. Find unbounded queries, missing indexes, inefficient includes/joins, non-selective filters, missing pagination, transaction contention, lock risks, and queries that will fail with large tenants.
```

```text
Design a caching strategy for this product. Include what to cache, what not to cache, tenant isolation, invalidation, staleness tolerance, cache stampede protection, security risks, and observability.
```

```text
Review frontend performance. Check bundle size, route-level loading, expensive components, memoization misuse, network waterfalls, image loading, virtualization needs, hydration cost, and slow form interactions.
```

```text
Create a cost and scalability review. Identify infrastructure costs that grow with users, data, jobs, AI calls, storage, logs, traces, and third-party APIs. Recommend limits, dashboards, and architectural changes before scale.
```
